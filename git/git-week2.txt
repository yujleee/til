핵심 쏙쏙 Git 2주차 정리

0. 협업의 단계
1) 이슈 할당 : 새로운 작업이나 버그 등 각 단위별로 작업자 배정
2) 브랜치 생성, 작업 : 할당받은 이슈를 해결하기 위한 브랜치를 생성하여 작업!
		이 때, 브랜치 이름을 기능별로 만들면 프로젝트 관리에 좋다.
3) 커밋 : 작업 후 브랜치에 커밋
4) 머지 : 브랜치 작업 내역 기본 브랜치(main 같은)에 병합하는 과정
4-1) PR후 머지 : 필요에 따라 작업내역 PR 후 머지

1. Issue?
해야할 작업, 에러, 버그 리포트 등등
이슈를 해결함으로써 프로젝트에 기여한다!

2. branch
맡은 이슈에 대해 작업하는 공간


3. Merge conflict
병합 충돌! 머지하는 과정에서 같은 파일의 동일한 부분이 수정되면 발생한다.
충돌 내역을 수정해서 다시 커밋하여 해결!


4. tracking
트래킹은 사실 원격 repo의 트래킹된 브랜치와 연결해주는 것!
기본적으로 트래킹되고 있는 브랜치를 기준으로 push하고 pull해오는 것이다.
