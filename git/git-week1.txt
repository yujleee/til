2. 1주차 주요 개념 키워드 적어보기
- 버전관리 : 프로젝트의 변경 상태를 정리해서 관리하는 것
- git init : 컴퓨터에 있던 프로젝트를 로컬 repo로 변경
- add / staging : 커밋할 파일을 스테이지에 올려두기
- commit : 현재 프로젝트의 변경사항을 스냅샷처럼 저장! Add -> 커밋메세지 작성 -> commit의 과정을 거침
- history : 프로젝트 변경 내역
- repository : git으로 관리되는 프로젝트
- push : 로컬 repo의 커밋들을 원격 repo에 반영하기
- pull : 원격 repo의 커밋들을 로컬 repo로 반영하기 
- clone : 원격 repo를 내 컴퓨터로 가져와기 (복제)
